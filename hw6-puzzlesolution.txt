HW6 Puzzle Solution

Part 1: Proof of Work

I wrote a Python script using hashlib to brute force a nonce such that:
 
sha256(cnet_id + nonce)

was numerically less than:
0000000fffffffffffffffffffffffffffffffffffffffffffffffffffffffff

For cnet_id = jcruiz, I found:

nonce = 79563878
hash  = 0000000b1093090909f01b60127730f14fd6dbaac9a60375bdf801cafc1dcf6a

This required iterating through nonces until the hash condition was satisfied,
which only took a few minutes of computing.

Part 2 – PUZZLE

The file PUZZLE contained SHA256 hashes of:

    sha256( key(9-digit string) + token )

where tokens come from TEXT.split().

Step 1: Recover 9-digit key

After many test interations of creating Python scripts that ran for far too long
and overheated my computer, I decided to write a multithreaded C program using CommonCrypto 
SHA256 to brute-force all 9-digit keys (000000000–999999999).

Instead of checking full dictionaries, I tested each candidate key against a small set of 
common English tokens ("a", "I", "to", "of", "the", "and", etc.), and checking whether any
generated hash matched one in the puzzle.

This found:

puzzle_key = 747585522

Step 3: Find misspelled word

By comparing the expected tokenization of the correct quote against PUZZLE hashes,
I located exactly one mismatch at index 62, which had an intended word of 'circumstances.'

I then generated edit-distance candidates and found that the PUZZLE token was:

cirupmstances

This is the single misspelled word in the PUZZLE file.

In total, including all the test attempts and computing time, this task took around 4-5 hours.
